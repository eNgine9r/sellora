# Pilot first-run guide

## Path 1 — Почати зі своїми даними
1. Create or select the real QA workspace.
2. Open Dashboard and review the first-run checklist.
3. Configure workspace settings.
4. Add the first product and variant.
5. Add stock through Inventory.
6. Create a lead or customer.
7. Create the first order and confirm Dashboard progress updates.

## Path 2 — Переглянути демо Sellora
1. Select `Переглянути демо Sellora`.
2. Confirm a separate `Демо Sellora` workspace appears.
3. Verify the demo banner explains synthetic data.
4. Open Leads, Customers, Products, Inventory, Orders, Finance, Advertising and Analytics.
5. Switch back to the real workspace and confirm no DEMO records appear.
6. Use `Прибрати демо-простір` when finished.

## Roles
- OWNER: full first-run checklist and demo management.
- MANAGER: operational checklist only; no protected workspace settings or demo deactivation.
- ANALYST: read-only orientation; no create/edit/archive CTA.

## Deferred scope
Public signup, billing, live Meta/Instagram sync, real Nova Poshta TTN creation, import deep QA, finance deep QA and advertising deep QA remain separate sprints.
