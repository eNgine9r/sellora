# Пілотний onboarding guide Sellora

Sellora — це MVP CRM/ERP для Instagram-магазинів в Україні. Мета пілоту: перевірити, чи допомагає Sellora пройти шлях від Instagram-ліда до клієнта, замовлення, відправлення, прибутку та повторного продажу.

## Для кого цей пілот

- Невеликі Instagram-магазини, які зараз ведуть товари, замовлення, клієнтів або рекламу в таблицях.
- Команди, яким потрібно бачити продажі, залишки, рекламні витрати й базову аналітику в одному місці.
- Власники, які готові дати фідбек по реальних процесах, але не передавати приватні дані в документацію або скриншоти.

## Що протестувати першими кроками

1. Увійдіть у demo workspace або створіть безпечний тестовий простір.
2. Перегляньте Dashboard: дохід, замовлення, ROAS, топ товарів, залишки.
3. Відкрийте Products та Inventory, перевірте чи зрозумілі картки/таблиці товарів.
4. Відкрийте Orders і подивіться статуси, позиції замовлення та оплату.
5. Відкрийте Analytics, порівняйте звіти з очікуваннями магазину.
6. Запустіть dry-run імпорту на копії або синтетичному файлі, а не на приватному оригіналі.

## Як додати demo або реальні дані

- Для демонстрації використовуйте `backend/scripts_seed_demo.py`, який створює лише DEMO записи.
- Для реального магазину спочатку зробіть копію таблиці, видаліть приватні зайві поля, запустіть dry-run і перевірте попередження.
- Не додавайте реальні телефони, приватні замовлення або рекламні кабінети в issues, docs, скриншоти чи тестові файли.

## Як імпортувати каталог

1. Відкрийте `/settings/import`.
2. Оберіть preset Product catalog.
3. Завантажте файл і натисніть Preview.
4. Перевірте мапінг колонок: назва/SKU товару, SKU варіанта або колір/розмір, ціна, залишок, мінімум, категорія, фото.
5. Запустіть Validate і Dry-run.
6. Якщо лічильники коректні й немає критичних errors, виконайте імпорт.
7. Перевірте `/products`, `/inventory`, `/analytics`.

## Як імпортувати історію замовлень

1. Оберіть preset Historical orders.
2. Перевірте колонки: order number, date, customer, phone або Instagram, variant SKU, quantity, price, cost, status, payment, tracking.
3. За замовчуванням historical import не має змінювати поточний склад (`affect_inventory=false`).
4. Повторені order number мають згрупуватися в одне multi-item order.
5. Після імпорту перевірте Dashboard, Sales report, Product report і Customer report.

## Як додати рекламні метрики

- Оберіть Advertising history або додайте метрики вручну у `/advertising`.
- Мінімальні поля: date, campaign, platform, spend, messages/leads/orders, revenue.
- Meta Ads API ще не підключено; це ручні або імпортовані історичні метрики.

## Як читати Dashboard

- Revenue показує дохід за вибраний період без скасованих/повернених замовлень.
- Net Profit доступний тільки ролям з фінансовим доступом.
- ROAS показує дохід від реклами на кожну витрачену гривню.
- Low stock alerts показують товари, які потребують уваги.

## Як повідомляти проблеми

Для кожної проблеми вкажіть:

- сторінку;
- що ви робили;
- очікуваний результат;
- фактичний результат;
- чи проблема блокує роботу;
- без приватних телефонів, реальних замовлень, workspace IDs або токенів.

## Що ще не автоматизовано

- Instagram Direct API ще не підключено.
- Meta Ads API ще не підключено.
- Billing/subscriptions ще немає.
- Nova Poshta потребує реальних налаштувань/API key і додаткової staging-перевірки.
- Повний production onboarding ще буде розширюватися після пілоту.

## Як надсилати фідбек у пілоті

Натисніть **Фідбек** у верхній панелі Sellora, оберіть категорію, за бажанням поставте оцінку 1–5 і коротко опишіть проблему або ідею. Не додавайте паролі, API-ключі, токени, workspace IDs або приватні дані клієнтів у повідомлення.
